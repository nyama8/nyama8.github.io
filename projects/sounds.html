sound-engine

This is a simple set of Python modules that are capable of representing and playing music. The program consists of a simple hierarchy of classes representing different aspects of a musical piece. Through the classes and methods defined in these modules, one can create music using simple synthesized sounds (similar to retro 8-bit music).

The data structures used to represent the music also present the possibility for straightforward integration with new functions for the creation of sheet music and other visualization tools.

Object hierarchy

  • Melody: An object that represents a segment of a larger musical piece with well-defined tempo and number of beats.
    • Voice: An object representing a single "voice" within the Melody to which it belongs. The voice is a assigned a specific instrument and consists of a single string of (typically non-overlapping) notes.
      • Note: An object that represents a single note within a voice, but has well-defined properties that are transferable to other voices.
        • FmNote: A specific instance of the Note class that enables the frequency/pitch of the note to be adjusted in time.
          • Vibrato
          • Grace
          • Drop
          • Glissando
        • PercNote: An object representing percussion instruments with short duration (such as some drums and cymbals).
        • LongPercNote: An object representing percussion instruments with longer durations.
  • Song: A object wrapper that combines multiple Melody objects together to form a larger total song. This no longer stores the exact notes as the Melody object would, but does contain references to each constituent Melody object.
  • instruments.py: A module containing different instruments and note envelope profiles that voices and notes use to generate waveforms represented as function objects. Including the following and others:
    • sine
    • triangle
    • square
